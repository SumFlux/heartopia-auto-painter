# Heartopia 自动画画脚本

读取 [转换器](../converter/README.md) 导出的像素矩阵 JSON，自动在心动小镇游戏中选色、点击绘制。

## 功能

- GUI 控制面板（PySide6）
- 画布 + 调色板坐标标定（共 6 次 Enter，数据自动保存）
- 按颜色分组批量绘制 + 蛇形遍历（最小化鼠标移动）
- 快捷键控制（F5 开始 / F6 暂停恢复 / F7 停止）
- 4 档速度调节（20ms ~ 200ms）
- 断点续画（中断后可从上次位置继续）
- 无需管理员权限

## 环境要求

- **Windows 系统**（依赖 Win32 API 查找游戏窗口）
- **Python 3.8+**
- 心动小镇游戏正在运行，且已进入画画界面

## 安装

```bash
pip install -r requirements.txt
```

依赖：pynput、pywin32、Pillow、PySide6、NumPy。

## 启动

```bash
python auto_painter.py
```

或直接双击 `auto_painter_main.bat`（自动安装依赖并启动）。

## 使用流程

### 1. 导入像素数据

点击「导入 JSON 文件」，选择转换器导出的 `*_heartopia.json` 文件。

日志会显示：
- 矩阵尺寸（如 `100x100`）
- 需绘制的像素数
- 是否包含 `colorId`（精确定位标识）

### 2. 标定画布（2 次 Enter）

> 标定数据会自动保存到 `calibration.json`，下次启动自动加载，无需重复标定。

1. 点击「标定画板范围」
2. 切到游戏窗口，将鼠标移到**画布左上角第一格**的中心 → 按 **Enter**
3. 将鼠标移到**画布右下角最后一格**的中心 → 按 **Enter**

```
┌─────────────────────┐
│ ① ← 左上角第一格     │
│                     │
│                     │
│     右下角最后一格 → ②│
└─────────────────────┘
```

### 3. 标定调色板（4 次 Enter）

1. 点击「标定调色板」
2. 将鼠标移到色系标签栏**最左侧可见的组** → 按 **Enter**
3. 将鼠标移到色系标签栏**最右侧可见的组** → 按 **Enter**
4. 将鼠标移到色块区域**左上角第一格**中心 → 按 **Enter**
5. 将鼠标移到色块区域**右下角最后一格**中心 → 按 **Enter**

```
  ①            ②        ← 色系标签栏（左侧/右侧）
┌────┬────┐
│ ③  │    │              ← 色块区域 2列×5行
├────┼────┤
│    │    │
├────┼────┤
│    │    │
├────┼────┤
│    │    │
├────┼────┤
│    │ ④  │
└────┴────┘
```

> 脚本会根据 ③④ 两个角点自动计算出全部 10 个色块的坐标。

### 4. 开始画画

1. 选择速度（建议先用「慢速」测试）
2. 点击「开始画画」或按 **F5**
3. **不要移动鼠标**，脚本会自动控制

绘画过程中：
- 进度条实时更新
- 颜色预览框显示当前绘制的颜色
- 日志显示当前颜色组和进度

### 5. 暂停 / 停止 / 续画

| 快捷键 | 功能 |
|:------:|------|
| **F5** | 开始画画 |
| **F6** | 暂停 ↔ 恢复（交替） |
| **F7** | 停止（进度自动保存） |

**断点续画**：F7 停止后，重新导入同一 JSON，点击「断点续画」按钮即可从中断处继续。

## 速度预设

| 预设 | 每次点击间隔 | 说明 |
|------|:----------:|------|
| 极慢 | 200ms | 最安全，适合首次测试 |
| 慢速 | 100ms | 稳定可靠 |
| 正常 | 50ms | 推荐日常使用 |
| 快速 | 20ms | 适合小尺寸画布 |

> 游戏切换颜色时固定等待 350ms（不受速度设置影响），这是游戏 UI 的响应瓶颈。

## 文件说明

| 文件 | 作用 |
|------|------|
| `auto_painter.py` | GUI 主程序入口 |
| `paint_engine.py` | 绘画引擎：颜色分组、蛇形遍历、断点续画 |
| `palette_navigator.py` | 调色板导航：翻页、选色 |
| `canvas_locator.py` | 画布坐标映射：像素坐标 → 屏幕坐标 |
| `mouse_input.py` | 输入后端抽象层（PynputBackend / PostMessageBackend） |
| `window_manager.py` | 游戏窗口查找与置顶 |
| `config.py` | 配置常量（窗口参数、速度预设、共享调色板 re-export） |
| `calibration.json` | 标定数据（自动生成，可删除重新标定） |
| `paint_progress.json` | 断点续画进度（自动生成，画完自动清理） |

## 工作原理

```
导入 JSON
    ↓
按 colorId 将像素分组（如 "1-0" 的所有像素归为一组）
    ↓
组内蛇形排序（偶数行左→右，奇数行右→左）
    ↓
按组号升序排列（减少调色板翻页次数）
    ↓
对每个颜色组：
  1. 翻页到目标色系组
  2. 点击目标色块
  3. 逐个点击该颜色的所有画布像素
    ↓
全部完成 / 中断保存进度
```

## 已知限制

1. **画画期间鼠标被占用**：使用 pynput 物理鼠标模拟，画画时不能动鼠标
2. **多显示器可能偏移**：高 DPI 缩放或多显示器环境下坐标可能不准，需要重新标定
3. **游戏窗口不能被遮挡**：点击在屏幕绝对坐标上，窗口被覆盖会导致点击到其他程序
4. **切换颜色有固定延迟**：游戏 UI 响应需要 ~350ms，这是速度的瓶颈

## 实验性功能：PostMessage 后台投递

`mouse_input.py` 中包含 `PostMessageBackend`，可以向游戏窗口发送后台鼠标消息，理论上不需要移动物理鼠标。但心动小镇（Unity 引擎）是否响应 PostMessage 鼠标消息**尚未验证**。

如果你想尝试，在 `auto_painter.py` 中将：
```python
self.backend = PynputBackend()
```
改为：
```python
hwnd = find_game_window()
self.backend = PostMessageBackend(hwnd)
```

## 故障排查

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| 点击位置偏移 | DPI 缩放 / 多显示器 | 确保系统缩放 100%，或重新标定 |
| 鼠标飞到屏幕外 | 标定坐标错误 | 按 F7 停止，重新标定画布 |
| 颜色选错 | 调色板标定不准 | 重新标定调色板，确保标签和色块位置准确 |
| 游戏不响应点击 | 窗口失焦 | 确保游戏在最前，不要切换窗口 |
| 「未找到游戏」 | 游戏未运行或窗口标题不匹配 | 确保心动小镇正在运行 |
| 标定尺寸不匹配 | 换了不同尺寸的 JSON | 删除 `calibration.json` 重新标定 |
